# Анимации (описание для дизайнера)

Дизайнер предусматривает в макете **состояния и намерения движения**; разработчик реализует анимации на **Framer Motion** ([этап 3](./development-phases.md)).

Базовые стили компонентов — **[components.md](./components.md)**; тени и цвета — **[design.md](./design.md)**.

---

## Появление при загрузке

| Элемент | Эффект |
|---------|--------|
| **Hero** — заголовок и кнопки | **fade-up:** opacity `0 → 1`, смещение по **Y** `20px → 0` |

---

## При скролле

| Сценарий | Поведение |
|----------|-----------|
| **Блоки** при попадании во вьюпорт | **fade-up** (аналогично по смыслу загрузочному эффекту) |
| **Карточки внутри блока** | Появление **по очереди**: первая сразу, вторая через **0,1 с**, третья через **0,2 с** и т.д. (stagger **~100 ms**) |

---

## При наведении (hover)

| Элемент | Анимация |
|---------|----------|
| **Карточка товара** | Масштаб **1 → 1.03**, тень **усиливается** |
| **Кнопка Primary** | Масштаб **1 → 1.02**, тень **усиливается** |
| **Иконка маркетплейса** | Цвет **`#666666` → `#E87A00`** |

**Согласование с вёрсткой:** в [components.md § 3.5](./components.md#35-карточка-товара) для карточки товара также задан подъём **`translateY(-4px)`** и усиление тени — в реализации допустимо **совместить** лёгкий подъём и `scale` до **1.03** (или унифицировать с макетом).

---

## Интерактивные (циклические / по событию)

| Элемент | Поведение |
|---------|-----------|
| **Кнопка CTA** | **Пульсация:** `scale` **1 → 1.05 → 1**, цикл каждые **5 с** |
| **Счётчики** (например «500+ проектов») | Число **плавно** нарастает от **0** до целевого значения |
| **Бургер-меню** | Три полоски **трансформируются в крест** (см. [components.md § 3.9](./components.md#39-бургер-меню-мобильная-версия)) |
