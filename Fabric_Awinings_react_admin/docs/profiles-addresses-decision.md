# Профили клиентов и адреса доставки в Staff API / React Admin

**Решение (2026-04):** отдельные ресурсы `customer-profiles` и `shipping-addresses` в `/api/staff/v1/` **в этом репозитории не внедрялись** — операции по ним остаются в **Django Admin (Unfold)** до отдельного ТЗ.

**Обоснование:** в спецификации ([admin-react-admin-migration-analysis.md §10](./admin-react-admin-migration-analysis.md)) они указаны как «минимум полей» и PДн; приоритет отдан каталогу, настройкам сайта, главной и сервисным actions. При необходимости отдельного эпика: ViewSet + сериализаторы + RA-ресурсы + тесты 401/403 по образцу существующих staff-ресурсов.
