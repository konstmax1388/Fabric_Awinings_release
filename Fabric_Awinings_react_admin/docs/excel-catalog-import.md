# Импорт каталога из «фабрика тентов контент.xlsx»

Таблица от заказчика: один лист, первая строка — заголовки. Ниже — **реальные строки с товарами** (на момент выгрузки в репозиторий их **14**; остальные ~980 строк листа пустые — зарезервировано под расширение).

**Яндекс.Диск:** колонка с URL на диск **не используется** при импорте (фотографии подключатся отдельно позже).

---

## 1. Структура колонок (маппинг)

Индексы — от нуля, как при чтении строки массивом.

| Индекс | Заголовок в файле | Назначение при импорте |
|--------|-------------------|-------------------------|
| 0 | Остатки | Число: справочно; в Django можно положить в `stock` / `quantity` **до** синка с Битрикс24 или игнорировать, если источник остатков только Б24. |
| 1 | Категория | Текст: создать/найти **Category** (slug из транслита или словарь соответствий). В файле встречались: «Тенты для бассейнов», «Аксессуары для банных печей», «Чехлы для мебели». |
| 2 | (пусто) | Пропуск. |
| 3 | Название | **Основное название товара** (`Product.name`); обязательное поле для строки импорта. |
| 4 | Контент | URL Яндекс.Диска — **не импортировать** (заглушка изображений или пропуск). |
| 5 | (число, большой ID) | Стабильный **внешний идентификатор** для идемпотентности (например `external_id` / `bitrix_xml_id` / поле `sku` если согласовано с Б24). У разных товаров значения различаются — удобно как ключ **update-or-create**. |
| 6 | (короткий текст) | Краткое описание / подзаголовок для карточки или SEO (`short_description` / часть `description`). |
| 7 | (часто пусто) | Резерв; при появлении смысла — уточнить у заказчика. |
| 8 | Цвет | Атрибут: строка в `description` / отдельное поле `color` / свойство в JSON — как заведено в модели. |
| 9 | Материал | **Материал** для фильтра каталога (`material` / связь с справочником). |
| 10 | Диаметр (см) | Число: характеристика (для тентов бассейнов); в описание или поле `diameter_cm`. |
| 11 | Характеристики | Свободный текст; в `description` или JSON. |
| 12 | WB | URL карточки на **Wildberries** → `marketplace_links.wildberries` (или ключ в вашей модели). |
| 13 | OZON | URL **Ozon** → `marketplace_links.ozon`. |
| 14 | MARKET | URL **Яндекс Маркет** (если есть). |
| 15 | Мвидео | Опционально, если используется на сайте. |
| 16 | Детский мир | Опционально. |
| 17 | комм алиса | Опционально (внутренние пометки заказчика — можно не выводить на витрину). |
| 18+ | Пустые | Игнорировать. |

**Примечание:** в некоторых строках часть ссылок МП пустая — импорт должен переносить только непустые URL, без падения на `null`.

## 2. Рекомендуемая стратегия импорта

1. **Формат:** management-команда Django, например `python manage.py import_catalog_xlsx path/to/file.xlsx` (или CSV-экспорт из Excel для простоты diff в Git — по желанию).

2. **Идемпотентность:** ключ строки — **колонка 5** (числовой ID), при отсутствии — составной ключ `(категория, нормализованное название)` с логированием предупреждений.

3. **Пропуск строк:** если колонка 3 (название) пустая — строка не товар.

4. **Категории:** нормализация названий (trim); при новой категории — создать запись с уникальным `slug`.

5. **Цена:** в выгрузке **нет** отдельной колонки цены — цену задаёт **Битрикс24** после синка или вручную в админке до подключения Б24; при импорте можно ставить `0` / placeholder и помечать «нужна цена из Б24».

6. **Изображения:** не тянуть с Яндекс.Диска; после импорта — массовая загрузка файлов в медиа или сопоставление по `external_id`.

7. **Кодировка и типы:** читать через **openpyxl**; числа Excel (остатки, диаметр) — приводить к `int`/`Decimal` осознанно; URL — строки, обрезка пробелов.

8. **Отчёт:** в консоль — сколько создано/обновлено/пропущено; список строк с ошибками валидации.

9. **Согласование с ТЗ 2.0:** после запуска магазина с **Б24** импорт из Excel остаётся полезен для **миграции** или **черновика**; «истина» по номенклатуре и остаткам — в CRM. Имеет смысл хранить **внешний ID** из колонки 5 в поле, совместимом с маппингом на товар Б24.

---

## 3. Проверка качества данных

- Дубликаты названий в одной категории — предупреждение.
- Неконсистентные URL (не http/https) — отбрасывать или исправлять.
- Опечатки в категориях — словарь синонимов перед созданием slug.
