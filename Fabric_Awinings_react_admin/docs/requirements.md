## 3. Окружение разработки (Docker)

**Цель:** на время разработки поднимать **отдельные контейнеры Docker**, не мешая другим проектам на том же ПК, где уже используется Docker.

**Принципы (к реализации при старте работ):**

| Область | Что сделать |
|---------|-------------|
| Имя проекта Compose | Уникальное имя (`COMPOSE_PROJECT_NAME` или `name:` в compose-файле), чтобы префиксы контейнеров/сетей/томов не пересекались с другими стеками. |
| Порты на хосте | Явно заданные значения (желательно через `.env`), без конфликта с уже занятыми портами других сервисов. |
| Сеть | Изолированная сеть в рамках compose-проекта (стандартное поведение Compose при уникальном имени проекта). |
| Тома | Именованные тома с префиксом/областью проекта, чтобы данные не смешивались с другими контейнерами. |

Детали образов, сервисов и портов — по мере появления в ТЗ.

**Frontend dev:** Vite на порту **17300** (и локально, и в контейнере), чтобы не пересекаться с типичным **5173** других проектов; в Docker маппинг **17300:17300**, переопределение — `FABRIC_FRONTEND_PORT` в `.env`, см. **README.md**.

## 5. Продакшен: целевой хостинг

В будущем сайт планируется развернуть на хостинге со следующим составом ПО (версии — как у провайдера, Ubuntu 24.04). Для разработки приложения напрямую важны прежде всего **веб-стек**, **PHP** и **MySQL**; остальное — инфраструктура площадки (почта, DNS, квоты, безопасность).

| Компонент | Версия / примечание |

| Apache | MPM-ITK |
| PHP | Composer; модуль PHP; **PHP-FPM** |
| Nginx | `1:1.26.3-96-ubuntu24` |
| PageSpeed Module for Nginx | `1:1.26.3-96-ubuntu24` (пакет привязан к версии nginx) |
| awstats | `7.9-1` |
