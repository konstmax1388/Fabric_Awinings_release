## Замечания по плану

- Этапы **4–8** взаимосвязаны: без Б24/СДЭК/эквайринга «полный магазин» по ТЗ не закрыт.
- Импорт чернового каталога из Excel — [excel-catalog-import.md](./excel-catalog-import.md); не заменяет синхронизацию с Б24.
- Стек в ТЗ: в документе указан **PostgreSQL**; **на проде проекта — MySQL 8.0.45** (см. [requirements.md](./requirements.md)). Также **Axios**, **React Hook Form + Yup**, **Swiper**, **Zustand/RTK** — при необходимости выровнять с текущим кодом (TS, fetch и т.д.).

## Статус реализации (код) — деталь

- **Этап 1** — сделано: Vite/React/Tailwind, **Django + DRF** (`backend/`), **CORS**, `docker compose` (`api` + `frontend`), **React Router**, `VITE_API_URL`, `GET /api/health/`.
- **Этап 2** — главная: **11 блоков** в `frontend/src` (`SiteHeader` … `SiteFooter`), страницы-заглушки `/catalog`, `/portfolio`, `/contacts`, `/blog`.
- **Этап 3** — **Framer Motion** на главной: пресеты в `frontend/src/lib/motion-presets.ts`, `AnimatedCounter` / `PulsingCTA`, fade-up + stagger по секциям, hover карточек, анимация смены цены в калькуляторе, бургер и мобильное меню в `SiteHeader` (см. [BACKLOG.md](../BACKLOG.md)).
- **Этап 4 (часть)** — каталог: `CatalogPage` / `ProductPage` — данные с **`GET /api/products/`** (фильтр `?category=`, сортировка, пагинация `?page=`); корзина **`/cart`**, оформление **`/checkout`** (шаги контакты → доставка → подтверждение), `POST /api/leads/cart/`. **Формы:** маска телефона РФ, валидация имя/email/комментария, honeypot `website`, throttle лидов на API. Нет: СДЭК, оплаты.
- **Этап 5 (часть)** — ЛК: маршруты `/account/*` (lazy), регистрация/вход, смена пароля, профиль, адреса доставки, заказы; связка гостевого заказа с аккаунтом по email — по логике бэкенда. Нет: трек СДЭК в ЛК, «повтор заказа».
- **Маркетплейсы** — у товара поле `marketplaceLinks`; `MarketplaceLinks` с `hrefById` и `linkKeys`; глобальные URL — `GLOBAL_MARKETPLACE_URLS` в `site.ts`.
- **Калькулятор** — `lib/calculator.ts`, отправка заявки **`POST /api/leads/calculator/`**; форма с теми же правилами валидации/антиспама, что и остальные лиды.
- **Обратный звонок / карта** — `HeroCallbackModal`, `MapFormSection` (заявка с `leadSource: other`), `POST /api/leads/callback/`.
- **Backend** — модели и миграции в `backend/api/`, JWT, `seed_demo`.
- **Админка** — Django `admin.py` (не React Admin).
- **SEO** — `react-helmet-async`, `public/robots.txt`, динамический **`/sitemap.xml`** (Django, см. `config/urls.py`, `api/views_seo.py`); для прода задать `DJANGO_PUBLIC_SITE_URL`.
- **Фронт: производительность** — ленивые чанки страниц (`React.lazy` в `App.tsx`), компонент оптимизированных изображений и API вариантов.
- **Версионирование** — файл **`VERSION`** в корне репозитория; в админке Unfold отображается номер версии (см. `config/version.py`, `unfold_environment_callback`).
- **Тесты** — `pytest` в `backend/tests/` (в т.ч. `test_seo_and_images.py`, расширенный `test_api.py` для лидов).

### Маршрутизация и корзина (важно для отладки)

- В **`App.tsx`** layout **`AppShell`**: `CartProvider` → `<Outlet />` — все страницы внутри **`RouterProvider`**, иначе `Link` вне контекста ломает SPA.
- Корзина: страница **`/cart`** (`CartPage`, `components/cart/CartView.tsx`), оформление заказа **`POST /api/leads/cart/`**.

### Чек-лист ручного теста

1. `docker compose up --build`, дождаться API и Vite.
2. `docker compose exec api python manage.py seed_demo` (если каталог пустой).
3. Главная: подборка, портфолио, отзывы, превью блога с API.
4. Каталог: фильтр, сортировка, пагинация; карточка товара и «похожие».
5. **`/cart`** и **`/checkout`**: мультистеп, валидация контактов, оформление — номер заказа и подтверждение с сервера.
6. Калькулятор и форма на главной/карте: отправка — 201 и запись в админке (колбэк / калькулятор).
7. ЛК: регистрация, вход, профиль (smoke).
8. `GET /sitemap.xml` на API — не 500, корректный XML.
9. `npm run build` в `frontend/`.
