# ТЗ 2.0 (апрель 2026): выжимка и согласование с документацией

Источник: файл заказчика **`ТЗ2.0.docx`** (версия 2.0, финальная). Этот документ фиксирует **смысловые изменения** относительно прежней версии требований в репозитории (ТЗ 1.0: лендинг + каталог + админка) и **расхождения с текущим кодом**.

**Безопасность:** в приложении к ТЗ у заказчика могут быть логины/пароли API (СДЭК, банк и т.д.). Их **не копируют** в Git и в общие документы — только переменные окружения и защищённое хранилище.

---

## 1. Что изменилось по сути

| Область | Было в docs v1 | ТЗ 2.0 |
|--------|----------------|--------|
| Тип проекта | Landing + каталог + админка | **Полноценный интернет-магазин** (каталог, корзина, ЛК) **+ лендинг** |
| Каталог и остатки | Локальное управление товарами в админке | **Источник истины — Битрикс24** (REST API). На сайте: **кэш**, отображение; **локально не ведётся** полный CRUD товаров |
| Админка — товары | Полный CRUD (название, цена, фото…) | Только **показ/скрытие**, **порядок** в каталоге, **SEO** (meta), не перезаписываемые из Б24 |
| Заказы | Заявки, упрощённая корзина (`CartOrder`) | Оформление **→ Битрикс24** (`sale.order.add` и аналоги по факту API), **вебхуки** статусов (`OnSaleOrderStatusChange`), локальный флаг «не синхронизирован» при ошибках |
| Доставка | Не зафиксирована | **СДЭК API v2**: тарифы, ПВЗ на **Яндекс.Картах**, создание отправления, вебхуки статусов, печатные формы |
| Оплата | Не зафиксирована | **Эквайринг**: карты (Visa, MC, МИР), **СБП**; **двухстадийный** платёж; виджет; фискализация 54-ФЗ; возвраты. В ТЗ упомянуты материалы **Ozon Acquiring** — провайдер уточняется при внедрении |
| Оплата при получении | — | **Наложенный платёж через модуль СДЭК** (и связка с эквайрингом/Ozon — по инструкциям провайдера) |
| Личный кабинет | Не было в v1 | История заказов, статусы, **трек СДЭК**, **повтор заказа**, профиль и адреса |
| Оформление заказа | Упрощённо | **Многошагово:** контакты → доставка (СДЭК, ПВЗ на карте) → оплата → подтверждение/оплата |
| Маркетплейсы | 4 иконки | **Wildberries, OZON, Яндекс Маркет, Авито** — в хедере/футере и **персональные ссылки** в карточке товара; управление в админке |
| Хедер | Без уточнения | **Липкий** (sticky) |
| Каталог | Фильтры категория, материал | + сортировка **по популярности**; пагинация **или** бесконечная подгрузка |
| Карточка товара | «Заказать» | **Счётчик количества**, **«Добавить в корзину»** |
| Стек (ТЗ) | Общий | **PostgreSQL**, **Celery + Redis**, **React Admin** для админки, **Axios**, **React Hook Form + Yup**, **Swiper**, **Zustand или Redux Toolkit** и др. (см. §3) |
| Фоновые задачи | Не описаны | Периодический **синк каталога/остатков**, fallback статусов, **повтор** несинхронизированных заказов, очистка кэша (таблица в ТЗ) |
| Сроки | Не в docs | **10–14 недель**, 1 разработчик full-time |
| Этапы в ТЗ | — | **12 этапов** (от настройки проекта до тестов интеграций) — см. [development-phases.md](./development-phases.md) |

## 2. Нефункциональные требования (кратко)

- **SEO:** Schema.org (в т.ч. Product, LocalBusiness, Order, Offer), OG, lazy loading, Core Web Vitals (LCP, FID, CLS — пороги в ТЗ).
- **Безопасность:** CORS, CSP, rate limiting, HTTPS, резервное копирование БД и медиа.
- **Дизайн:** палитра и типографика **заданы числами** в ТЗ (`#FEFBF5`, `#E87A00`, `#2D5A27`, Playfair Display + Inter и т.д.) — должны совпасть с [design.md](./design.md) / [typography.md](./typography.md) при сверке.
- **Инфраструктура:** VDS Ubuntu, **Nginx + Gunicorn**, опционально Sentry, Prometheus/Grafana.

## 3. Расхождения с текущим репозиторием

| Тема | Код / доки сейчас | Целевое по ТЗ 2.0 |
|------|-------------------|-------------------|
| БД | SQLite в dev, модели товаров «свои» | **PostgreSQL** в проде; каталог с сайта — **реплика/кэш** полей из Б24 + overlay (видимость, порядок, SEO) |
| Админка | Django admin, полный CRUD товаров | **React Admin** (ТЗ) или по согласованию; для товаров — только поля overlay + просмотр синка |
| Корзина / заказ | Есть страница корзины и `POST /api/leads/cart/` | Полный **checkout**, оплата, **СДЭК**, создание заказа в **Б24** |
| ЛК | Нет отдельного ЛК покупателя | История, статусы, трек, повтор заказа |
| Фронт | TypeScript, fetch, свои формы | ТЗ допускает **JS + JSDoc** (TS опционально); **Axios**, **RHF + Yup**, **Swiper** — при желании выровнять со временем |
| Celery | Нет | Нужны **очереди** для синка и ретраев |

Импорт первичного контента из таблицы заказчика описан в **[excel-catalog-import.md](./excel-catalog-import.md)** — это **не** замена синхронизации каталога с Б24, а способ **начального наполнения** или черновика до подключения CRM.
