# Цели и функциональные требования

Документ фиксирует цели проекта, пользовательскую часть (**frontend**) и **административную панель** по ТЗ. Версия по согласованию с `requirements.md`.

---

## 1. Цели и задачи проекта

### 1.1. Основная цель

Создать современный, продающий, SEO-оптимизированный сайт для производства тентов на заказ, вызывающий доверие и желание оставить заявку.

### 1.2. Задачи

- Представить услуги компании в **B2B** и **B2C** сегментах.
- Обеспечить возможность **полного управления контентом** через админку.
- Интегрировать **ссылки на маркетплейсы** (Wildberries, OZON, Яндекс Маркет, Авито).
- Обеспечить **адаптивность (mobile first)** и **плавные современные анимации**.

---

## 2. Функциональные требования

### 2.1. Пользовательская часть (Frontend)

#### 2.1.1. Главная страница (лендинг)

Обязательные блоки — **в строгой последовательности**:

**Блок 1. Хедер**

- Логотип, меню навигации: **Главная**, **Каталог**, **Портфолио**, **Контакты**.
- Телефон для связи.
- Кнопка «Заказать расчёт».
- Иконки маркетплейсов: Wildberries, OZON, Яндекс Маркет, Авито.

**Блок 2. Hero (первый экран)**

- Заголовок H1, подзаголовок, призывная кнопка (CTA).
- Фоновое видео или изображение.
- Опционально — 3D-мокап готового изделия.

**Блок 3. Проблемы → Решения**

- От **3 до 6** карточек.
- Каждая карточка: проблема клиента (например, «Дорого?»), решение компании (например, «Своё производство — на 30% дешевле»), иконка.

**Блок 4. Виды тентов**

- Карусель или сетка категорий товаров.
- Каждая категория: название, иконка, изображение, ссылка на соответствующий раздел каталога.

**Блок 5. Калькулятор цены**

- Интерактивный блок.
- Поля: длина, ширина; выбор материала из выпадающего списка; чекбоксы дополнительных опций (люверсы, усиленные швы, карманы и т.д.).
- Цена пересчитывается **в реальном времени** с **плавной анимацией** отображения.
- Кнопка отправки заявки.

**Блок 6. Портфолио**

- Плитка проектов с **фильтрацией по категориям**.
- Каждый проект: фотография (до/после), заголовок, категория, дата.

**Блок 7. Почему мы**

- От **3 до 5** колонок с преимуществами.
- Каждая колонка: иконка, заголовок, краткое описание.

**Блок 8. Отзывы**

- Слайдер (карусель) с отзывами.
- Каждый отзыв: фото клиента, имя, текст, рейтинг звёздами; опционально — ссылка на видеоотзыв (превью с иконкой play).

**Блок 9. Блог**

- Три последние статьи из блога.
- Карточка: изображение, дата, заголовок, краткое описание, ссылка «Читать далее».

**Блок 10. Карта + форма захвата**

- **Яндекс Карта** с меткой местоположения производства (слева или справа — по макету).
- Форма обратной связи: имя, телефон, комментарий; кнопка отправки.

**Блок 11. Футер**

- Логотип и краткое описание компании.
- Навигационное меню.
- Иконки-ссылки на маркетплейсы (Wildberries, OZON, Яндекс Маркет, Авито).
- Ссылки на социальные сети.
- Контакты: телефон, email, физический адрес.
- Копирайт и ссылка на политику конфиденциальности.

#### 2.1.2. Страница каталога

- Сетка товаров в виде карточек: фото, название, цена, кнопка «Заказать».
- Фильтры: боковая панель или выпадающие — по **категориям**, по **материалам**.
- Сортировка: по цене (возрастание/убывание), по новизне.
- **Пагинация** или **бесконечная подгрузка** при скролле.

#### 2.1.3. Карточка товара

- Фотогалерея (карусель).
- Характеристики: размеры, материал, доступные опции.
- Кнопка «Заказать».
- Блок ссылок на маркетплейсы: четыре кнопки с иконками (Wildberries, OZON, Яндекс Маркет, Авито) — каждая ведёт на **страницу этого же товара** на платформе.
- Блок **похожих товаров** (рекомендации).

#### 2.1.4. Страница портфолио

- Плитка проектов с фильтром по категориям и **пагинацией**.
- По клику — детальный просмотр: крупные фото, описание.

#### 2.1.5. Страница контактов

- Схема проезда (**Яндекс Карта**).
- Юридические реквизиты, физический адрес, график работы, телефон, email.
- Форма обратной связи.

#### 2.1.6. Страница блога

- Список всех статей с **пагинацией**.
- Страница статьи: полное содержимое (**WYSIWYG**-форматирование, изображения, видео).

### 2.2. Административная панель (Admin Panel)

**Общий принцип:** каждый текстовый, медийный или настраиваемый элемент сайта должен редактироваться **через админку** без правки кода.

#### 2.2.1. Управление контентом главной страницы

Для каждого блока главной администратор редактирует указанные поля.

| Блок / сущность | Редактируемое |
|-----------------|---------------|
| **Hero** | Заголовок, подзаголовок, текст кнопки, ссылка кнопки, фоновый файл (видео или изображение). |
| **Проблемы → Решения** | Заголовок блока; список карточек (проблема, решение, иконка). Карточки: добавление, удаление, **смена порядка**. |
| **Виды тентов** | Заголовок блока; список категорий (название, иконка, изображение, ссылка). Категории: добавление, удаление, **смена порядка**. |
| **Калькулятор** | Заголовок блока; материалы (название, цена за м², флаг «по умолчанию»); опции (название, цена, единица измерения); текст кнопки отправки. |
| **Портфолио** (блок на главной) | Заголовок блока; проекты (фото до/после, заголовок, категория, дата). Проекты: добавление, удаление, редактирование. |
| **Почему мы** | Заголовок блока; карточки (иконка, заголовок, описание). Карточки: добавление, удаление, **смена порядка**. |
| **Отзывы** | Заголовок блока; отзывы (фото клиента, имя, текст, рейтинг 1–5, опционально ссылка на видео). Отзывы: добавление, удаление, **смена порядка**. |
| **Футер** | Текст копирайта; соцсети (название, иконка, ссылка); email, телефон, адрес. |
| **Маркетплейсы** | Общие ссылки на магазины компании: Wildberries, OZON, Яндекс Маркет, Авито; **переключатель видимости** всего блока. |

#### 2.2.2. Управление товарами (каталог)

Для каждого товара:

- Название и **slug** (ЧПУ).
- Артикул.
- **Категория** — связь с категориями из блока «Виды тентов».
- Материал.
- Размеры: длина, ширина; опционально высота.
- Базовая цена.
- **Фотографии** — несколько файлов, **смена порядка**.
- Четыре поля ссылок на товар: Wildberries, OZON, Яндекс Маркет, Авито.
- **SEO:** meta title, meta description, keywords.
- **Статус:** опубликован / черновик.

#### 2.2.3. Управление статьями блога

Для каждой статьи:

- Заголовок и slug.
- Мета-описание для ПС.
- Полный текст — **WYSIWYG** (изображения, видео, форматирование).
- Главное изображение.
- Дата публикации.
- Имя автора.
- **Статус:** опубликована / черновик.

#### 2.2.4. Глобальные настройки сайта

- Логотип сайта и **favicon**.
- Контактный телефон, email, режим работы.
- Ссылки **WhatsApp** и **Telegram** для быстрой связи.
- Глобальные **meta title** и **meta description** (по умолчанию, если не заданы на странице).
- Коды счётчиков: **Яндекс.Метрика**, **Google Analytics**.
- Код **hCaptcha** или **reCaptcha** (защита форм).

#### 2.2.5. Управление заявками

Администратор может:

- Просматривать все заявки из **калькулятора** и **форм**.
- Фильтровать по **дате** и **статусу** (новая / обработана).
- Менять статус заявки.
- **Экспорт** в CSV или Excel.
- **Email-уведомления** при поступлении новой заявки.

#### 2.2.6. Управление пользователями админки

**Роли:**

| Роль | Доступ |
|------|--------|
| **Администратор (admin)** | Полный доступ ко всем разделам админки. |
| **Менеджер (manager)** | Только **заявки** и **блог**. |

Аутентификация: **email** и **пароль**.

### 2.3. Интеграции

#### Маркетплейсы (Wildberries, OZON, Яндекс Маркет, Авито)

Два уровня:

1. **Общие ссылки** на магазины компании — в **футере** сайта; те же URL из админки используются для иконок в **хедере** (§2.1.1). Настройка: §2.2.1 «Маркетплейсы».
2. **Индивидуальные ссылки** на карточку конкретного товара на каждом маркетплейсе — в **карточке товара** на сайте (поля в §2.2.2).

#### Яндекс Карты

- Отображение **местоположения производства** на **главной** (блок «Карта + форма», §2.1.1) и на **странице контактов** (§2.1.5).

#### Яндекс.Метрика и Google Analytics

- Сбор статистики посещений.
- Отслеживание **событий**: клики по кнопкам, отправка форм (детализация целей — при настройке и внедрении).

#### Email-уведомления

- Автоматическая отправка **содержимого заявки** на указанный **email заказчика** (администратора сайта).

#### Защита форм

- **hCaptcha** или **reCaptcha** — защита от спама (код и выбор провайдера задаются в глобальных настройках, §2.2.4).
